🤖 BMO Agent — Arquitectura y Flujo

be-more-agent · Raspberry Pi 5 · Ollama + Piper + openwakeword

📋 1. Flujo Principal (Main Loop)

Trigger sources: Wake word openwakeword > umbral 0.5 · PTT (Enter key) · stdin CLI
Fallback: Si no hay modelo wake word → solo PTT

🔄 2. Máquina de Estados

Notas: Al entrar a IDLE se muestra DORMIDO primero · Cada 20s en IDLE → cara emocional aleatoria por 4s · CORAZON no se usa para expresar emoción, es la reacción inmediata al wake word
FPS animación: IDLE/emociones = 500ms · SPEAKING = 50ms (frames aleatorios)

🧠 3. Flujo de Decisión (LLM → Acción)

Detección de acción: El LLM output contiene { o "action": → modo acción · Si no → modo charla
Tool aliases: "google", "browser" → search_web · "look", "see" → capture_image · "cara", "emotion" → show_face
Fallback: Si la acción no es válida y el value tiene >1 palabra → CHAT_FALLBACK (lo trata como texto)
